# Breakfast POS UI/UX Design System

## 1. 設計定位

### 1.1 品牌目標
- 類型：早餐店收銀與營運管理系統
- 氣質：明亮、俐落、可信任、節奏快
- 核心印象：專業但不冰冷，適合高頻操作與日常營運

### 1.2 使用情境
- POS：收銀員在尖峰時段需要用最少點擊完成點餐、結帳、送單
- KDS：廚房人員需要快速辨識新單、等待時間與備註
- Kiosk / QR：顧客要能在無人協助下自然完成點餐
- Admin：管理者要能在資訊密度高的情況下快速找到報表、設定與異常狀態

### 1.3 設計原則
- 高可讀：第一眼就看懂商品、金額、狀態與下一步
- 高回饋：每個操作都要有視覺或文字結果
- 高一致：同類元件在三端保持一致語意
- 高容錯：資料載入中、失敗、空狀態都不能是白屏
- 高效率：尖峰場景優先，非必要操作不增加點擊層級

## 2. 視覺風格

### 2.1 主視覺方向
- 底色：以白底與淺霧藍背景建立乾淨現代感
- 面板：大量使用白色卡片面板與柔和陰影
- 強調色：品牌藍用於主要 CTA、數字焦點、關鍵標籤
- 氣氛：不要科技黑，也不要過度醫療感，要像高級早餐品牌的營運工具

### 2.2 設計關鍵字
- Clean
- Fast
- Bright
- Operational
- Friendly

## 3. 色彩系統

### 3.1 品牌色
- `brand-700`：`#1E40AF`
- `brand-600`：`#1D4ED8`
- `brand-500`：`#2563EB`
- `brand-400`：`#60A5FA`
- `brand-100`：`#DBEAFE`
- `brand-50`：`#EFF6FF`

### 3.2 中性色
- `ink`：`#0F172A`
- `slate-700`：`#334155`
- `slate-500`：`#64748B`
- `slate-200`：`#E2E8F0`
- `slate-100`：`#F1F5F9`
- `surface`：`#F8FAFC`
- `white`：`#FFFFFF`

### 3.3 狀態色
- Success：`#16A34A`
- Warning：`#F59E0B`
- Danger：`#EF4444`
- Info：`#0EA5E9`

### 3.4 狀態色用法
- Success：下單成功、儲存成功、上線中
- Warning：庫存低、等待過久、資料待確認
- Danger：刪除、取消、失敗、黑名單
- Info：通知、系統提示、外送平台狀態

### 3.5 背景用法
- 全站背景不使用純平面單色，採用淡藍霧面漸層
- Panel 永遠保持高亮白底，讓資料區塊清楚浮出
- Danger / Warning 背景採低飽和淡色，不要用高飽和紅黃大面積壓迫閱讀

## 4. 字體系統

### 4.1 字體選擇
- 中文：`Noto Sans TC`
- 數字 / 金額 / 訂單號：`JetBrains Mono`

### 4.2 字級建議
- Hero 標題：`32-40px`
- 區塊標題：`20-24px`
- 卡片標題：`16-20px`
- 內文：`14-16px`
- 提示文字：`12-13px`
- 金額大字：`28-40px`

### 4.3 字重規則
- 站點標題、總額、訂單號：`800-900`
- 區塊標題：`700-800`
- 一般表單與列表：`500-600`
- 輔助說明：`400-500`

## 5. 版面系統

### 5.1 間距規則
- 基礎單位：`4px`
- 常用間距：`8 / 12 / 16 / 20 / 24 / 32`
- 卡片內距：
  - 緊湊資料卡：`16-20`
  - 主要工作卡：`20-24`
  - Hero 面板：`28-40`

### 5.2 圓角
- 主面板：`24-28px`
- 次面板：`20-24px`
- 輸入框 / 按鈕：`16-20px`
- 小標籤 / pill：`999px`

### 5.3 陰影
- 主面板：柔和大陰影，建立「浮起來」但不厚重的層次
- 子卡片：短距離輕陰影，避免多重面板時視覺太亂

## 6. 元件語言

### 6.1 Primary Button
- 用途：登入、結帳、儲存、送出
- 風格：品牌藍底、白字、厚實高度
- 高度：
  - 桌機：`44-52px`
  - Kiosk / 觸控：`56-72px`
- Hover：加深一階
- Active：`scale(0.98)`
- Disabled：降低對比但保留位置

### 6.2 Secondary Button
- 白底灰框
- 用於返回、切換、補印、次要編輯

### 6.3 Danger Button
- 淡紅底或紅色邊框
- 僅用於刪除、取消、停用

### 6.4 Input / Select / Textarea
- 白底、淡灰邊框、Focus 時出現品牌藍 ring
- 輸入框高度一致
- 金額與數量欄位強制使用等寬字體

### 6.5 Pill / Badge
- 用於分類、狀態、桌號、套餐、外送平台
- 不同狀態顏色應一致：
  - `PENDING`：淺藍
  - `PREPARING`：淺橘
  - `READY`：淺綠
  - `CANCELLED`：淺紅

### 6.6 Cards
- 商品卡：圖示、名稱、價格、描述
- 訂單卡：訂單號、時間、品項摘要、狀態、備註
- 指標卡：標題、數值、補充說明

### 6.7 Empty / Loading / Error States
- Empty：使用輕量說明文字與引導，不可只有空白
- Loading：骨架屏或明確文案
- Error：置於內容區內顯示，而不是整頁中斷

## 7. 動畫系統

### 7.1 原則
- 動效是回饋，不是裝飾
- 快速系統不要用大幅拖慢操作的動畫

### 7.2 建議節奏
- Hover / Press：`120-180ms`
- 面板浮現：`220-280ms`
- Modal 進場：`180-240ms`
- Toast：`220ms` 進、`180ms` 出

### 7.3 建議動畫
- 卡片 hover：微上浮 `translateY(-2 ~ -4px)`
- CTA click：輕壓縮 `scale(0.98)`
- Modal：`opacity + translateY(8px)` 進場
- 頁面首屏：主標題、卡片 stagger reveal
- KDS 新單：卡片邊框 pulse 一次
- Caller 最新叫號：大字 toast 淡入並微放大

### 7.4 禁止事項
- 不要過度使用 bounce
- 不要長時間旋轉 loading
- 不要讓列表更新造成整頁晃動

## 8. 各模組 UI 規劃

### 8.1 POS

#### 版型
- 桌機：`菜單區 / 購物車區`
- iPad 橫向：維持雙欄
- iPad 直向：菜單在上、購物車在下

#### 菜單區
- 分類 tab 固定在上方，可橫向滑動
- 商品卡採 2-4 欄依螢幕寬度切換
- 售完商品灰化並加 badge
- 時段特價顯示原價刪除線

#### 購物車區
- 上方顯示總品數與清空鍵
- 品項卡需明確顯示：
  - 商品名
  - 加料摘要
  - 備註
  - 單價
  - 數量調整
- 底部固定顯示：
  - 小計
  - 折扣 / 點數折抵
  - 合計
  - 主要結帳按鈕

#### 結帳流程
- Modal 分左右兩區：
  - 左：訂單類型、桌號、會員、備註
  - 右：付款方式、收款、找零、確認
- 結帳成功後要出現成功確認視窗
- 成功視窗需包含：
  - 訂單號
  - 訂單金額
  - 訂單類型
  - 收到「送單成功」的明確訊號

### 8.2 KDS
- 採大卡片網格
- 卡片資訊順序：
  - 訂單號
  - 經過時間
  - 類型 / 桌號
  - 品項清單
  - 備註
  - 操作按鈕
- 逾時色彩：
  - 10 分鐘：橘色外框
  - 15 分鐘：紅色外框
- 完成後觸發大字 toast 與叫號事件

### 8.3 Caller
- 最新叫號區要最大
- 製作中號碼放左右或下方列表
- 歷史叫號維持次要層級
- 背景可更乾淨，避免干擾遠距離閱讀

### 8.4 Kiosk
- 首頁只保留兩個大選項：內用 / 外帶
- 商品卡要更大、更少字、更偏觸控導向
- 結帳步驟用 step 感清楚呈現
- 60 秒閒置回首頁需搭配倒數提示

### 8.5 QR 點餐
- 與 Kiosk 共用大部分視覺
- 需明確顯示桌號與「本桌加點」語意
- 若偵測到既有訂單，畫面中要顯示「將併入本桌現有訂單」

### 8.6 Admin

#### 導航
- 左側固定導覽列
- 選中項目需高對比與明顯背景
- 管理資訊較多，導覽標籤用詞要直白

#### Dashboard
- 第一排：三個核心 KPI
- 第二排：每小時訂單量 + 低庫存
- 第三排：營業額趨勢
- KPI 卡要更強調數值，不要平均分散注意力

#### 菜單管理
- 左側建立 / 編輯表單
- 右側列表與匯入匯出
- 圖片、SKU、分類、價格、成本、庫存屬於第一層欄位
- 套餐 JSON 與時段價格 JSON 屬於進階設定，可視覺分區

#### 報表
- 用折線、長條、熱力圖區分不同資訊
- 匯出按鈕固定於頁首操作區

## 9. 響應式規範

### 9.1 Desktop
- 使用完整資訊密度
- 保留雙欄 / 三欄工作流

### 9.2 iPad
- 觸控區最小尺寸 `44px`
- 側欄可轉為上方 tab 或抽屜
- 表格避免太寬，必要時轉卡片顯示

### 9.3 Small Tablet
- 卡片單欄堆疊
- 固定底部主要按鈕區

## 10. 可用性與資安互動

### 10.1 可用性
- 高頻操作按鈕永遠在預期位置
- 關鍵操作需顯示成功 / 失敗
- 載入狀態不可阻止已載入內容閱讀

### 10.2 權限可視化
- STAFF 看不到不屬於自己的管理項
- 無權限按鈕應直接隱藏或 disabled，避免點擊後才報錯

### 10.3 資安體驗
- 後台登入錯誤訊息不要透露帳號是否存在
- Session 過期後要柔性導回登入，不可停留空白頁

## 11. 目前系統可優化項目

### 11.1 POS
- 訂單快覽可再加入狀態色條，減少閱讀負擔
- 成功彈窗可加入「列印中 / 已送廚 / 可繼續點餐」的分層提示

### 11.2 Admin
- Dashboard KPI 卡可加入日同比或目標差距
- 菜單管理進階欄位可改折疊區，降低第一次使用壓力

### 11.3 KDS / Caller
- 可補入更明顯的聲音設定與大字模式切換

## 12. 建議實作順序

1. 先整理 design tokens：顏色、陰影、圓角、字級、狀態色
2. 再統一元件：button、input、badge、panel、table、modal、toast
3. 優先重做 POS、KDS、Admin Dashboard 三個最核心頁
4. 補 Kiosk / QR 的觸控體驗與 idle flow
5. 最後針對動畫、微互動與空狀態進行細修

## 13. 驗收標準

- 每個頁面都有 loading / empty / error 狀態
- 主要 CTA 一眼可辨識
- 收銀流程 3 步內完成送單
- KDS 新單可在 1 秒內辨識
- Admin 首屏 5 秒內可看懂今日營運狀態
- iPad 上主要操作不需縮放即可完成
